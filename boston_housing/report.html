Implementation: Define a Performance Metric¶

It is difficult to measure the quality of a given model without quantifying its performance over training and testing. This is typically done using some type of performance metric, whether it is through calculating some type of error, the goodness of fit, or some other useful measurement. For this project, you will be calculating the coefficient of determination, R2, to quantify your model's performance. The coefficient of determination for a model is a useful statistic in regression analysis, as it often describes how "good" that model is at making predictions.

The values for R2 range from 0 to 1, which captures the percentage of squared correlation between the predicted and actual values of the target variable. A model with an R2 of 0 is no better than a model that always predicts the mean of the target variable, whereas a model with an R2 of 1 perfectly predicts the target variable. Any value between 0 and 1 indicates what percentage of the target variable, using this model, can be explained by the features. A model can be given a negative R2 as well, which indicates that the model is arbitrarily worse than one that always predicts the mean of the target variable.

  • How it can be applied to optimize a learning algorithm?

Hint: When explaining the Grid Search technique, be sure to touch upon why it is used, what the 'grid' entails and what the end goal of this method is. To solidify your answer, you can also give an example of a parameter in a model that can be optimized using this approach.

Answer: Almost all models have some parameters that cannot be learned from the training data itslef, but have to be manually set up. They are usually called hyper-parameters, and with a grid search what we want is to find their optimum values in order to obtain the best possible model. We can manually try different values for these hyper-parameters, but if the list of hyper-parameters is long and the range of values they can take is large, it would be exhausted. With the grid search technique we are brute-forcing all posible combinations of the hyper-parameters to find the optimum one.

For example, let's pretent we are fiting the data using the C-Support Vector Classification model implemented in sklearn.svm.SVC. We are not sure about what values should we use for 'C', 'degree' and 'coef0' hyper-parameters once we decided to use the poly kernel. We can create a three dimensional grid, i.e. a 3D discrete space, with all possible combinations we want to try. For example, we want to try 'C' from 0.1 to 1.0 in 0.05 steps, 'degree' from 2 to 5 and 'coef0' from 0.0 to 10.0 in 0.5 steps.

The main problem with this technique is that we will have to fit a lot of different models, one for each combinations of the hyper-parameters. In this case 19x4x21=1596 different models. This can take a lot of time!

Question 8 - Cross-Validation¶

  • What is the k-fold cross-validation training technique?

  • What benefit does this technique provide for grid search when optimizing a model?

Hint: When explaining the k-fold cross validation technique, be sure to touch upon what 'k' is, how the dataset is split into different parts for training and testing and the number of times it is run based on the 'k' value.

When thinking about how k-fold cross validation helps grid search, think about the main drawbacks of grid search which are hinged upon using a particular subset of data for training or testing and how k-fold cv could help alleviate that. You can refer to the docs for your answer.

Answer: If we tune the hyper-parameters of a model based on the scores we get from the test subset, we can overfit the model and perhaps the hyper-parameters we chose works great in that particular test subset but don't generalize well. To avoid that problem, instead of spliting the data in two subsets: train and test; we can split in three subsets: train, validation and test. We would use the train subset to fit the model, the validation subset to tune the hyper-parameters (or choose the best model), and the test subset to check if those values generalize well or if we have overfitted the model. The problem is that spliting data into three groups leave us with less data to train the model and we have already seen that due to "the curse of dimensionality" the more complex our model is (more features for example) the more data we need, so if we start saving more and more data to testing purposes we'll be in trouble.

k-fold cross-validation training technique allows us both to maximize the data we can use to train the model and validate the model to avoid overfitting. k-fold CV splits the training subset in k sub-subsets. k-1 of them are used to train the model and the left one is used to validation. This process is repeated k times so at the end we have trained the model with the k different combinations of (k-1) sub-subsets and have validated with all the k splits. For each time we train the model we obtain an error score computed from the validation sub-subset. The final score for this particular model is the mean of all these values.

Again, this technique is computationally expensive but maximizes the available data to train the model. The grid search implemented in sklearn.model_selection.GridSearchCV has a 'cv' parameter that if we set to an integer, it would be the k in the k-fold technique. This means that we can find the optimum values for the hyper-parameters of the model validating in 'cv'='k' different validation sub-subsets of the training subset.

To sum up, first we split the data in two subsets: training and testing (80%/20% or 75%/25%, typically). Then we split the training subset in k sub-subsets. We use cross validations as explained before to choose the best model, and finally we test the model with the testing subset, which contains data that the model has never seen before.

from sklearn.tree import DecisionTreeRegressor
from sklearn.metrics import make_scorer
from sklearn.model_selection import GridSearchCV, ShuffleSplit

def fit_model(X, y):
    """ Performs grid search over the 'max_depth' parameter for a 
        decision tree regressor trained on the input data [X, y]. """
    
    # Create cross-validation sets from the training data
    # sklearn version 0.18: ShuffleSplit(n_splits=10, test_size=0.1, train_size=None, random_state=None)
    # sklearn versiin 0.17: ShuffleSplit(n, n_iter=10, test_size=0.1, train_size=None, random_state=None)
    #cv_sets = ShuffleSplit(X.shape[0], n_iter = 10, test_size = 0.20, random_state = 0)
    cv_sets = ShuffleSplit(n_splits=10, test_size=0.2, random_state=0)

    # TODO: Create a decision tree regressor object
    regressor = DecisionTreeRegressor()

    # TODO: Create a dictionary for the parameter 'max_depth' with a range from 1 to 10
    params = {'max_depth': [i for i in range(1, 11)]}

    # TODO: Transform 'performance_metric' into a scoring function using 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric)

    # TODO: Create the grid search cv object --> GridSearchCV()
    # Make sure to include the right parameters in the object:
    # (estimator, param_grid, scoring, cv) which have values 'regressor', 'params', 'scoring_fnc', and 'cv_sets' respectively.
    grid = GridSearchCV(estimator=regressor, param_grid=params, scoring=scoring_fnc, cv=cv_sets)

    # Fit the grid search object to the data to compute the optimal model
    grid = grid.fit(X, y)

    # Return the optimal model after fitting the data
    return grid.best_estimator_
